# परिचय

मार्कडाउन और Mdx पार्सिंग unified और अन्य remark और rehype पैकेजों के माध्यम से समर्थित है। next-mdx-remote हमें .mdx और .md फाइलों को एक अधिक लचीले तरीके से पार्स करने की अनुमति देता है, बिना webpack को छुए।

GitHub फ्लेवर मार्कडाउन का उपयोग किया गया है। mdx-prism कोड ब्लॉक्स के लिए सिंटैक्स हाइलाइटिंग की सुविधाएँ प्रदान करता है। यहाँ यह दिखाने के लिए एक डेमो है कि सब कुछ कैसे दिखता है।

निम्नलिखित मार्कडाउन चीटशीट को इस स्रोत से अनुकूलित किया गया है: https://guides.github.com/features/mastering-markdown/

# मार्कडाउन क्या है?

मार्कडाउन एक तरीका है जो वेब पर टेक्स्ट को स्टाइल करने के लिए उपयोग किया जाता है। आप दस्तावेज़ का प्रदर्शन नियंत्रित करते हैं; शब्दों को बोल्ड या इटैलिक के रूप में फॉर्मेट करना, चित्र जोड़ना, और सूचियाँ बनाना कुछ ऐसी चीजें हैं जो हम मार्कडाउन के साथ कर सकते हैं। ज्यादातर, मार्कडाउन बस सामान्य टेक्स्ट होता है जिसमें कुछ गैर-अक्षरीय प्रतीक होते हैं, जैसे # या *।

# सिंटैक्स गाइड

यहां एक अवलोकन है कि आप GitHub.com या अपनी खुद की टेक्स्ट फाइलों में कहीं भी मार्कडाउन सिंटैक्स का उपयोग कैसे कर सकते हैं।

## हेडर

```
# यह एक h1 टैग है

## यह एक h2 टैग है

#### यह एक h4 टैग है
```

# यह एक h1 टैग है

## यह एक h2 टैग है

#### यह एक h4 टैग है

```
_यह टेक्स्ट इटैलिक होगा_

**यह टेक्स्ट बोल्ड होगा**

_आप **इन्हें** संयोजित कर सकते हैं_
```

_यह टेक्स्ट इटैलिक होगा_

## सिंटैक्स हाइलाइटिंग

यहां एक उदाहरण है कि आप GitHub फ्लेवर मार्कडाउन के साथ सिंटैक्स हाइलाइटिंग का उपयोग कैसे कर सकते हैं: [GitHub फ्लेवर मार्कडाउन](https://help.github.com/articles/basic-writing-and-formatting-syntax/):

````
```js:fancyAlert.js
function fancyAlert(arg) {
  if (arg) {
    $.facebox({ div: '#foo' })
  }
}
```
````

और यह इस तरह दिखता है - खूबसूरती से रंगीन और स्टाइल किए गए कोड शीर्षकों के साथ!

```js:fancyAlert.js
function fancyAlert(arg) {
  if (arg) {
    $.facebox({ div: '#foo' })
  }
}
```

## तालिकाएँ 

आप शब्दों की सूची बनाकर और उन्हें हाइफन - (पहली पंक्ति के लिए) से विभाजित करके और फिर प्रत्येक कॉलम को पाइप | से अलग करके तालिकाएँ बना सकते हैं:

```
| पहला हेडर                    | दूसरा हेडर                     |
| --------------------------- | ---------------------------- |
| सेल 1 से सामग्री                | सेल 2 से सामग्री                |
| पहले कॉलम में सामग्री            | दूसरे कॉलम में सामग्री            |
```

| पहला हेडर                    | दूसरा हेडर                     |
