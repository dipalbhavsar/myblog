# परिचय

टेलविंड स्टार्टअर ब्लॉग में Next.js के इन-बिल्ट इमेज घटक का आउट-ऑफ-द-बॉक्स समर्थन है और यह स्वचालित रूप से मार्कडाउन या MDX दस्तावेज़ों में डिफ़ॉल्ट इमेज टैग को बदलकर प्रदान किए गए Image घटक का उपयोग करता है।

# उपयोग

नई पेज रूट / जावास्क्रिप्ट फाइल में इसका उपयोग करने के लिए, बस इमेज घटक को आयात करें और इसे इस प्रकार कॉल करें:

```js
import Image from 'next/image'

function Home() {
  return (
    <>
      <h1>मेरी होमपेज</h1>
      <Image src="/me.png" alt="Picture of the author" width={500} height={500} />
      <p>मेरी होमपेज पर स्वागत है!</p>
    </>
  )
}

export default Home
```

मार्कडाउन फाइल के लिए, आप डिफ़ॉल्ट इमेज टैग का उपयोग कर सकते हैं और बिल्ड प्रक्रिया में डिफ़ॉल्ट img टैग को Image घटक से बदल दिया जाएगा।

मान लीजिए कि हमारे पास data/img/ocean.jpg में एक फाइल है, तो निम्नलिखित कोड लाइन ऑप्टिमाइज़ की गई इमेज उत्पन्न करेगा।

```
![ocean](/static/images/ocean.jpg)
```

इसके अलावा, चूंकि हम MDX का उपयोग कर रहे हैं, हम सीधे इमेज घटक का उपयोग कर सकते हैं! ध्यान दें कि आपको एक निश्चित चौड़ाई और ऊंचाई प्रदान करनी होगी। img टैग पद्धति स्वचालित रूप से आयामों का विश्लेषण करती है।

```js
<Image alt="ocean" src="/static/images/ocean.jpg" width={256} height={128} />
```

_नोट_: यदि आप इमेज को सहेजने की कोशिश करते हैं, तो यह वेबपी फॉर्मेट में होगी, अगर आपका ब्राउज़र इसे समर्थन करता है!

# लाभ

- वेबपी के साथ छोटी इमेज़ साइज (~JPEG से 30% छोटी)
- उत्तरदायी इमेज़ - उपयोगकर्ता के व्यूपोर्ट के आधार पर सही इमेज़ साइज भेजी जाती है
- लेज़ी लोडिंग - इमेज़ व्यूपोर्ट में स्क्रॉल होते ही लोड होती हैं
- से बचाता है [Cumulative Layout Shift](https://web.dev/cls/)
- निर्माण समय के बजाय अनुरोध पर ऑप्टिमाइजेशन - निर्माण समय में कोई वृद्धि नहीं होती!

# सीमाएँ

- next/image पर निर्भरता के कारण, जब तक आप Cloudinary या Imgix जैसे बाहरी इमेज CDN का उपयोग नहीं कर रहे हैं, यह व्यावहारिक रूप से Vercel के बिना होस्टिंग के लिए आवश्यक है। यह इस कारण है कि यह घटक एक सर्वरलेस फ़ंक्शन की तरह कार्य करता है जो एक अत्यधिक ऑप्टिमाइज़ किए गए इमेज CDN को कॉल करता है।

- यदि आप Vercel से बंधे नहीं रहना चाहते हैं, तो आप lib/mdx.js में remarkPlugins से imgToJsx को हटा सकते हैं। इससे डिफ़ॉल्ट img टैग का प्रतिस्थापन नहीं होगा।

- वैकल्पिक रूप से, आप इमेज ऑप्टिमाइजेशन के निर्माण समय में समर्थन की प्रतीक्षा कर सकते हैं। एक अलग पुस्तकालय, next-optimized-images ऐसा करता है, हालांकि इसमें वेबपैक के माध्यम से इमेजेज़ को ट्रांसफॉर्म करना आवश्यक होता है, जो यहाँ नहीं किया गया है।

- बाहरी लिंक से इमेजेज़ next/image से नहीं पास की जाती हैं।

- सभी इमेजेज़ को public फ़ोल्डर में संग्रहीत किया जाना चाहिए, जैसे कि /static/images/ocean.jpeg।
